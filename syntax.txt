   creating the version
1... git init--Create an empty Git repository or reinitialize an existing one
ex -->    Users\risqu\desktop\GIT-tutorial> git init

2....git status--It lets you see which changes have been staged, which haven't, and which files aren't being tracked by Git. 
ex--> PS C:\Users\risqu\desktop\GIT-tutorial> git status
On branch master
No commits yet

3.. git add command adds new or changed files in your working directory 
example syntax-- git add . which is all current files will be add

PS C:\Users\risqu\desktop\GIT-tutorial>  git add .

Changes to be committed:
  (use "git rm --cached <file>..." to unstage)

        new file:   config.js
        new file:   syntax.txt
